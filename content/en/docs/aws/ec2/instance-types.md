# EC2 實例類型選擇指南

## 常見的實例類型

### 通用型 (General Purpose)
- **t3.micro** - 適合輕量級工作負載
- **m5.large** - 平衡的運算、記憶體和網路資源

### 運算優化 (Compute Optimized) 
- **c5.large** - 高效能處理器，適合 CPU 密集型應用

### 記憶體優化 (Memory Optimized)
- **r5.large** - 記憶體密集型應用

## 如何選擇

1. **評估工作負載需求**
2. **考慮成本效益**
3. **監控效能指標**
4. **適時調整大小**

## 實際經驗

在選擇實例類型時，建議從小規格開始，根據實際使用情況逐步調整。
